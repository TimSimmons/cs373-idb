                        Player

Contains static information about a player.

The Player model serves as somewhat of a parent class and container to the Player_Year model. Each Player model contains information about an individual player which remains persistent throughout a player's career. For example, a player's name, school and handedness are contained in this model.

The Player model has a one to many relationship with the Player_Year model.
